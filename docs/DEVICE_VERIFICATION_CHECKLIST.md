# 実機確認チェックリスト（ゲームマップ）

実機・複数端末で試す前の確認用です。項目にチェックを付けながら進めてください。

## 事前

- [ ] 開発／ステージング用 Firebase プロジェクトに接続している（本番誤接続がない）
- [ ] 位置情報・Bluetooth（使用する場合）の権限を許可済み
- [ ] テスト用ルーム ID／ニックネームを共有済み（複数人テスト時）

## 起動 → タイトル

- [ ] 初回起動でも **起動アニメ**（世界観オーバーレイ）が約 1.4 秒以上見える
- [ ] その後 **図形ロゴが上に移動**し、タイトル画面へ自然にフェードする（真っ暗・フリーズで止まらない）
- [ ] タイトルで **図形ロゴ + ONI PIN 文字** のサイズが以前より大きすぎない
- [ ] タイトル右上の **起動音 ON/OFF** が保存される

## 世界観プロファイル（地図スタイル・演出）

各プロファイルで **準備 → 地図表示 → 試合開始** まで一度ずつ目視する。

- [ ] **Cyber Night**: ビネット＋二重スキャン＋ネオン帯のスイープが過剰でなく、地図の道路が読める
- [ ] **Urban Horror**: 赤黒ビネット、VHS ノイズ、危険時の脈動が気持ち悪すぎず操作可能
- [ ] **Pop City**: 明るいウォッシュ、写真ピン・バウンスフラッシュが楽しく見える
- [ ] **Stealth Tactical**: 低彩度のベール、ピンがズームアウト時に控えめで地図が見える
- [ ] **Magical / Astronomy**: 既存の演出が破綻していない（必要なら上記と同様に目視）

## HUD / パネル可読性

- [ ] **暗い地図**（ホラー・天文・サイバー等）で上部タイマー行・下部スキルパネルの文字が読める
- [ ] **フェーズ表示**（隠密逃走 / 告発解禁 / 残りN分）が試合進行に合っている
- [ ] **イベント履歴1行** — 告発解禁・脱落・告発成否が HUD に残る
- [ ] 一行 HUD：**「すべて」** で鬼情報・状態・コンディションが **途切れず** 横スクロールする
- [ ] HUD 設定：**鬼情報** トグルで一行・展開の鬼情報が消える（目アイコン非表示ボタンは無い）
- [ ] HUD 設定：**一行表示の内容** が折りたたみでき、**地図ピン** が上の方にある
- [ ] 暴露アラート（赤帯）が表示されたとき、文言とボタンがタップしやすい
- [ ] 折りたたみ HUD を開閉してもレイアウトが崩れない

## v2 導線（プリセット・ルール可視化）

- [ ] **ホスト** ルーム作成直後: お手軽 / 標準 / じっくり の3択（スキップ可）
- [ ] **準備画面**: 「この設定だと何分・何人向け」の一行サマリ
- [ ] **非ホスト ロビー**: 告発重み・時間・人数感のルール概要カード
- [ ] **スキル長押し**: 残り CD が試合中ピン留め表示される
- [ ] **ウェルカム「遊び方」**: 第二ゲーム（脱落後）のページがある

## ピン・ログ過多時（LOD / レイヤー）

- [ ] ズーム **アウト** 時、ギミック／痕跡／暴露マーカーが段階的に省略・遅延表示される（世界観ごとに過密感が違うことを確認）
- [ ] 上部 HUD の **地図レイヤー** から、安全地帯・痕跡・鬼情報などを切替えられる
- [ ] **Urban Horror** 初期状態で `ghostRough` がオフのとき、脱落者の粗位置が出ない（意図どおりか確認）
- [ ] **Stealth Tactical** 初期で痕跡・暴露・スキルマーカー等がオフのとき、必要情報だけで遊べるか

## オンライン（Firestore）

- [ ] **Android / iOS 実機またはエミュレータ**で試す（Windows / macOS / Linux のデスクトップだけでは `google-services.json` が効かず、dart-define か `firebase_options` が無いと参加に失敗しやすい）
- [ ] **iOS**: `ios/Runner/GoogleService-Info.plist` が正しい Firebase アプリのものか、Xcode の **Bundle Identifier** がコンソールの iOS アプリと一致するか（ヒューマンエラーで別プロジェクトの plist を置くと、接続は成功しても想定外の DB になる）
- [ ] **デバッグ時**: ルームロビーに表示される `DBG: Firebase projectId` が、Firestore ルールをデプロイしたプロジェクトと同じか
- [ ] ルーム参加後 **メンバー一覧に自分が表示**され、下の **「ゲーム画面へ」** が見える
- [ ] ルーム参加・ニック表示・ホスト表示が正しい
- [ ] ホストが試合開始すると、非ホストも同じエリア・ギミック密度で開始する
- [ ] **試合中止投票**: 賛成が過半数でホスト側が終了処理される（反対・未投票の扱いも確認）
- [ ] 切断・バックグラウンド復帰後もルーム状態が致命的に壊れない（既知の制限は `OPERATIONS_CHECKLIST` 等を参照）

## プレイエリア（編集・保存・適用・開始）

- [ ] **円**: 編集 →「エリアを保存」→ スロットに増えるが、**適用前**は地図上の試合用エリア（サマリー表示）が **変わらない**
- [ ] **ホスト**: スロットを選択 →「適用」→ サマリー・地図の円／多角形が **そのスロットと一致**
- [ ] **ホスト**: 適用後にアプリ再起動 → 前回「適用」した形が `play_area_v1` から復元される
- [ ] **オンライン非ホスト**: ホストが「適用」すると `lobby_play_area` で地図が同期される（試合開始前から一致する想定）
- [ ] **オンライン非ホスト**: 試合開始後もホストと同じエリア・ギミック位置になる
- [ ] **円・遠隔開始**: 保存中心から **80m 以上** 離れた状態でホストが開始 → トースト「中心を現在地に合わせました」＋ 円中心が現在地付近になる
- [ ] **多角形・遠隔開始**: エリア外にいるまま開始 → エリア外警告・暴露・（逃走者）90秒脱落が **意図どおり** か（自動寄せはない）
- [ ] **GeoJSON 取込**: 即座に地図に反映され、試合開始でその形が使われる（スロット「適用」を挟まなくてよい）
- [ ] **編集保存のみで「適用」忘れ** → 試合が **古いエリア** のまま始まらないか（運用ミス確認）

## 接近・捕獲（早見表どおりか）

- [ ] **パニック圏**（接触圏より外側・エリア連動）で **約6秒** → パニック HUD・**匿名痕跡**（約7秒間隔）。脱落なし
- [ ] 接触圏（エリア連動）で **3秒** 警告 → さらに留まると **約4秒** で「接触拘束」HUD（拘束時間はエリア連動）
- [ ] 拘束中に鬼 **12m** 以内 or BLE → 即捕獲
- [ ] **捕獲結界スキル**: 地図配置 → **24秒**・55m 円。拘束とは別 HUD 文言（「捕獲結界」）
- [ ] **オンライン**: 他端末の結界円半径が設置者と同じ（`fromSkill` 同期）
- [ ] 感染と拘束が **別タイマー** で動く（パニックだけでは即脱落しない）

## 告発・人狼（3人以上）

- [ ] 解禁後、有効施設が **1 + 陣取りボーナス** になっている（HUD に `+1` / `-1` 表示）
- [ ] **本鬼**が施設付近にいる間、その施設だけ告発できない
- [ ] 人狼: 鬼化トグル・HUDに **「強制まで」** と **「切替CD」** が別表示
- [ ] 人狼: 強制切替後は切替CDがやや長い。放置で姿が自動反転（通知なし）

## スキル・遊び方（v3）

- [ ] チュートリアルのスキルステップから「遊び方」→ 該当スキルカードへジャンプ
- [ ] リプレイ軌跡ラベルに「ローカル」が出ない（鬼軌跡は「鬼」など）
- [ ] ワールドギャラリー: 確定ボタンが世界ごとに英語（和風は日本語）／試聴チップは日本語

## 第二ゲーム（脱落後）

- [ ] 逃走者脱落 → **3秒オーバーレイ**（残響体の操作説明）が一度だけ出る
- [ ] 初回脱落時: 使える施設/カメラが **ハイライト**、使えないものが薄くなる
- [ ] 逃走者脱落 → **残響体**: ジャック端子・告発施設で陣取り（解禁後）
- [ ] 鬼脱落 → **復讐の鬼影**: 告発施設で妨害・監視カメラ停止（グレー表示）
- [ ] 他端末でも陣取り/妨害後に有効告発施設数が変わる

## スキル・ギミック（抜粋）

- [ ] GPS 追随・エリア外警告・セーフ残り回数が表示どおり動く
- [ ] よく使うスキル 1〜2 個の CD 表示が正しい

## 通話中プレイ（Host-Light）

- [ ] 試合開始前ヒントに「通話を開始してから ONI PIN を前面に」の文言がある
- [ ] Discord 通話しながら ONI PIN 前面で GPS・捕獲が動く
- [ ] LINE 通話しながら ONI PIN 前面で GPS・捕獲が動く
- [ ] 非ホストが 10 分通話（ホストは前面）— 時間切れ・脱落が破綻しない
- [ ] ホストが 10 分通話（非ホスト救済）— 時間切れで試合が終わる
- [ ] 鬼前面・逃走者 background で捕獲 → 逃走者復帰後に脱落が反映される
- [ ] ホスト background 中の時間切れ → 非ホストが終了できる

詳細: [host_light_multiplayer.md](./host_light_multiplayer.md)

## 終了後

- [ ] **HUD 設定やログを開いたまま** リザルトへ行っても、シートが閉じてリザルトが操作できる
- [ ] リザルト／ロビー復帰で操作が戻る
- [ ] クラッシュや ANR がない（発生したらログと手順をメモ）
