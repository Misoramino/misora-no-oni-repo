# BLE 近接（Proximity）

> 引き継ぎ索引: [HANDBOOK.md](./HANDBOOK.md) / コードとテスト: [CHANGE_MAP.md](./CHANGE_MAP.md) § G

アプリの「BLE で近接を補助」は、**お互いの端末が BLE の広告／スキャンを動かしているか**で挙動が変わります。

## 両方オン

- 双方がスキャンとアドバタイズを行えるため、**近接バンド（近い／遠い等）を推定しやすい**です。
- 省電力にしたい場合は、試合中だけオンにする運用が現実的です。

## 両方オフ

- BLE は使わず、**GPS とルームのハートビートのみ**で距離感を推定します。
- ビル内など GPS が不安定な場所では、近接の精度は下がりがちです。

## 片方だけオン

- **オン側**は相手を見つけようとしても、相手がアドバタイズしていなければ **検出できません**（ほぼ「オフ同士」に近いことが多いです）。
- 近接演出を安定させたい場合は、**参加者全員で BLE をオン**にするか、全員オフで GPS 運用に統一するのがおすすめです。

## 補足

- OS の Bluetooth 権限・省電力設定でスキャンが止まることがあります。
- 本機能はゲームルールの補助であり、**必須ではありません**。
