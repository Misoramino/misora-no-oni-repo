## 設計思想



**常時GPS共有ではなく、断片情報・心理戦・都市追跡ゲーム。**



- ライブ位置ピンは基本なし

- **名前付き暴露** / **匿名痕跡** / **テキスト情報** を織り交ぜて読む

- 逃走者も **情報を読んで勝つ**（告発・特化役割）

## 告発システム（3人以上）



### 施設



- 試合ごとに **3〜5 点**（`gimmickSeed` で全端末一致）。**有効数**は状況で 1 から増加（下表）

- 世界観別名称（例: Urban Horror=調査本部、Astronomy=**宇宙連合・地球支部**）



| 条件 | 有効施設 +1 |
|------|-------------|
| 常時 | 1（最低） |
| 脱落 1 人以上 | +1 |
| 経過 ≥ 試合時間の 60% | +1 |
| 脱落 2 人以上 | +1 |

（残響体の人数は現状未加算。将来: 逃走側残響体 vs 鬼側影が施設・情報を奪い合う設計案あり）



### 解禁



- **（脱落 1 人以上 かつ 5 分経過）** または **試合時間の 60%経過**

- ホストが `accusation_unlocked` を1回 publish（`activeSiteIndices` 付き・全員同期）



### 利用



- **逃走者のみ**。人狼・脱落者・鬼は不可

- 施設半径内でプレイヤー選択 → 確認 → 告発

- 正解（`accusedUid` が `PlayerRole.hunter`）→ **逃走者陣営即勝利**（`runnerWin` / `accusation_success`）

- 失敗 → **告発者即脱落**（残響体へ）+ その試合では再告発不可



### Firestore



- `player_eliminated` — 捕獲時

- `accusation_unlocked` — ホストのみ

- `accusation_attempt` — 告発者

- `accusation_failed` — ホスト（失敗時）

- 成功時はホスト `publishMatchEnd`



**2人戦では告発無効**（自動正解防止）。
