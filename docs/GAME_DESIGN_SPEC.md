> **更新:** 接近・捕獲の3系統整理、第二ゲーム（残響体／鬼影）、人狼、告発。  
> **コード:** `lib/game/game_config.dart`, `lib/game/skill_reference.dart`（装備スキル文案・数値表）, `lib/features/game_map/match/match_skill_tick_evaluator.dart`, `lib/game/accusation_logic.dart`

## 設計思想

**常時GPS共有ではなく、断片情報・心理戦・都市追跡ゲーム。**

- ライブ位置ピンは基本なし
- **名前付き暴露** / **匿名痕跡** / **テキスト情報** を織り交ぜて読む
- 逃走者も **情報を読んで勝つ**（告発・特化役割）
- 脱落後も **第二ゲーム**（残響体 vs 鬼側影）で試合に関与

## 用語（混同しやすいので厳密に）

| 用語 | 意味 | 誰が使う |
|------|------|----------|
| **鬼の接触圏** | 鬼に近づいたときの「至近」判定半径（プレイエリア連動・約35〜95m） | 鬼側の基本接近（全試合） |
| **接触拘束**（タッチロック） | 接触圏内 **4秒** → 拘束（**拘束円はエリア×10%、45〜110m**・持続は半径÷2.8m/s+6秒、22〜75秒） | 鬼の基本捕獲フロー |
| **感染** | 接触圏より外側の **感染環**（エリア×14%、58〜115m）に **6秒** → 約22秒の露出バフ | 鬼の基本接近（全試合） |
| **スキル「捕獲結界」** | 地図タップで円を設置。**逃走者・鬼どちらも装備可**。半径55m・**24秒** | 装備スキル（`SkillIds.captureZone`） |

**接触拘束とスキル捕獲結界は別物。** ランタイムは `lockZone*`（中心・拘束タイマー等）。接触拘束は `lockZoneFromSkill == false`、スキル結界は `true`。オンラインの `capture_zone_placed` は payload **`fromSkill: true`** で全員同期。

## プレイヤー向け早見表（鬼が近づいたとき）

**前提:** 逃走者がいる試合でのみ有効（鬼単独では感染・接触は動かない）。鬼の位置が分からない間は接触圏・感染の GPS 判定は走らない（テストモード除く）。

### 一覧（同時進行しうる）

| 系統 | 発動距離の目安 | 何秒で何が起きる | ペナルティ |
|------|----------------|------------------|------------|
| **① 接触圏 → 接触拘束** | **35〜95m**（エリア×8%） | **3秒**警告 → **4秒**で拘束開始 → 拘束 **22〜75秒**（拘束円半径÷2.8m/s+6秒） | 拘束中 **GPS 12m** or **BLE接触**で即捕獲。拘束円（エリア×10%、45〜110m）外 **10秒**で暴露→捕獲 |
| **② 感染（パニック）** | **58〜115m**（エリア×14%、接触圏より外側） | **6秒**連続で感染開始 → **22秒**感染 | **7秒ごと**匿名痕跡（偽位置中はデコイ近傍）。脱落なし |
| **③ スキル捕獲結界** | 設置円 **55m** | **24秒**で消滅。CD **80秒** | 効果中に円外 **10秒**で名前付き暴露→捕獲可なら脱落。**終了後**は結界消滅・旧位置から離れても暴露なし |

### タイムライン例（逃走者が鬼に近づき続けた場合）

```
距離
  │
感染環├──── 感染カウント開始（6秒で感染・距離はエリア連動）
  │
接触圏├──── 接触圏に入る（35〜95m目安）
  │          ├─ 3秒: 「接触圏内」警告
  │          └─ +4秒: 接触拘束（18秒）開始
  │
 0m└──────── BLE接触 or GPS 12m以内 → 即捕獲
```

**スキル結界**は鬼が近づいたからではなく、**自分または相手がスキルを使って地図に置いた円**が別途発動する。

### よくある誤解

| 誤解 | 実際 |
|------|------|
| 近づくと全部同時に終わる | 感染（露出）と接触拘束（捕獲リスク）は **別タイマー**。露出されてもすぐ脱落とは限らない |
| 捕獲結界＝鬼の接触圏 | **違う**。結界は **スキル**（逃走者・鬼どちらも可） |
| 拘束中は接触圏と同じ大きさ | 拘束円は **45〜110m**（エリア連動）。スキル結界は固定 **55m** |

## 接近・捕獲（3系統）

### 1. 鬼の接触圏 → 接触拘束 → 捕獲

```
逃走者が鬼の接触圏に入る
  → 3秒で警告（「鬼に近すぎます」）
  → さらに4秒留まると「接触拘束」（22〜75秒・エリア連動）
  → 拘束中: 拘束円（発動時の逃走者位置・半径45〜110m）内に留まる
  → GPS 12m または BLE contact で即捕獲
  → 拘束円外に10秒出ると: 位置暴露 → 捕獲（脱落）
  → 時間切れで拘束解除。至近だけでは再捕獲にならない
```

**接触拘束の円は逃走者端末のみ表示**（中心＝拘束開始時の逃走者位置。鬼に同期しない）。捕獲結界は設置円を範囲内の全員に表示。

**接触拘束解除後:** 捕獲結界を設置し直せば、結界の拘束中にごく至近で捕獲可能（接触圏4秒は不要）。

| 定数 | 値 | `GameConfig` |
|------|-----|--------------|
| 接触圏半径 | エリア半径×8%（35〜95m） | `scaledTouchRadius*` |
| 拘束まで | 4秒 | `touchLockRequiredSeconds` |
| 警告 | 3秒 | `touchLockRunnerNoticeSeconds` |
| 拘束時間 | **22〜75秒**（半径÷2.8m/s+6秒） | `MatchGeoHelpers.touchLockDurationSeconds` |
| 拘束円半径 | エリア×10%（45〜110m） | `scaledRestraintRadius*` |
| 直接捕獲距離 | **12m**（BLEは別） | `captureDistanceMeters` |

- **人狼の鬼化中**も同様の接近判定に近づく（`isHunterNow`）
- **逃走者0人**のときは感染・接触・捕獲をスキップ（鬼単独テスト除く）

### 2. 感染（接触圏とは独立）

| 定数 | 値 |
|------|-----|
| 発動距離 | エリア×14%（58〜115m） |
| 露出まで | 6秒（警告あり） |
| 感染時間 | 22秒 |
| 露出間隔 | 7秒 |

- **7秒ごと**匿名痕跡（偽位置中はデコイ近傍）
- 捕獲・拘束とは **別タイマー**

### 3. スキル「捕獲結界」（プレイヤー・鬼共通）

| 定数 | 値 |
|------|-----|
| 半径 | 55m |
| 持続 | **24秒** |
| CD | 80秒 |
| 範囲外10秒 | 効果中のみ名前付き暴露 → 捕獲可なら脱落 |
| 終了後 | 結界消滅・拘束解除。旧位置から離れても暴露なし |
| 接触拘束後 | 時間切れ解除後も結界で拘束し直し可。結界中の至近捕獲可 |

**逃走者:** 鬼を足止め・逆転トラップ。範囲内の全員（自分・味方・敵）を拘束。  
**鬼:** 逃走者を封じ込め・追い込み。オンラインではホストが `capture_zone_placed` / `capture_zone_bound` で同期。

**装備:** 逃走者・鬼・人狼（攪乱タイプ）が候補から選択可。文案・数値の一次ソースは `lib/game/skill_reference.dart`。

### プレイエリア外（逃走者のみ）

| 項目 | 値 |
|------|-----|
| 猶予 | 8秒（または中心から25m以内） |
| 再暴露 | 25秒ごと |
| 脱落 | **90秒**（接触拘束・スキル結界よりずっと長い） |

| 役職 | スキル候補 | 装備数 |
|------|------------|--------|
| 逃走者 | 偽位置、捕獲結界 | 1 |
| 鬼 | 偽情報暴露、捕獲結界、体投げ | 2 |

### 装備スキル早見（現行）

| スキル | 要点 |
|--------|------|
| **偽位置** | 約20秒。進行方向先にデコイ。名前付き・匿名・定期暴露はすべてデコイ近傍 |
| **偽情報暴露** | 自分 or ランダム逃走者の名前付き暴露を地図に設置（理由タグなし） |
| **体投げ** | 約90m以内に人形。回収まで捕獲・暴露の基準は人形。他スキル不可 |
| **捕獲結界** | 半径55m・24秒。全員拘束。効果中の円外離脱のみ暴露リスク |
| **鬼化・人化** | 自動切替 `min(15分, 試合÷2)`。切替可能までは間隔÷3。自動直後はやや延長。HUDで**自動切替まで**／**切替可能まで**を別表示 |

### 逃走者特化（3人以上）

| 特化 | 効果 |
|------|------|
| アナリスト | 匿名痕跡の時間帯・源・信頼度 |
| ハッカー | 情報屋距離帯精密化 + 鬼の方角（座標ピンなし） |

## 告発システム（3人以上）

### 施設

- 試合ごと **3〜5 点**（`gimmickSeed`）
- **有効数:** 解禁前 **0** → 解禁時 **1**
- 脱落数・経過時間だけでは **増えない**
- **陣取り:** 残響体の告発施設チャージで `territoryBonus +1`（試合2回）、復讐の鬼影の妨害で `-1`（試合3回）→ `activeAccusationSiteCount`（`accusation_sites.dart`）

### 解禁

- **（脱落1人以上 かつ 5分経過）** または **試合時間60%**
- ホストが `accusation_unlocked` を1回 publish

### 利用

- **生存逃走者のみ**（人狼・脱落者・鬼は不可）
- **生存中の本鬼**が施設半径内にいる間、その施設だけ告発不可（他施設は可）
- 正解 → 逃走者即勝利
- 失敗 → **告発者即脱落** → 残響体へ（`forEliminatedFaction` で鬼陣営は復讐の鬼影）

## 第二ゲーム：脱落後

| ルール | 陣営 | 主な行動 | 実装 |
|--------|------|----------|------|
| **残響体** | 逃走脱落（既定モード時） | カメラジャック（5回/試合）・告発陣取り（+1・2回/試合・解禁後） | ✅ |
| **復讐の鬼影** | 鬼脱落（既定モード時） | 告発妨害（−1・3回/試合）・カメラ停止（各カメラ1回・回数上限なし） | ✅ |
| 幽霊 | — | 観戦・粗位置 | ✅ |
| 鬼合流 | 鬼側 | 索敵支援（粗位置） | ✅ |

**分岐:** 試合の脱落後ルールが `spectralOperative`（残響体）のとき、**脱落時点の陣営**で分岐 — 人陣営→残響体、鬼陣営→復讐の鬼影（`EliminationAftermathRule.forEliminatedFaction`）。

### 残響体 vs 復讐の鬼影

| 行動 | 残響体 | 復讐の鬼影 |
|------|--------|------------|
| 監視カメラ | ジャック端子でチャージ→鬼位置暴露 | カメラごとに停止（1回限り・地図からグレー表示） |
| 告発施設 | 陣取りチャージ→ `territoryBonus +1` | 妨害チャージ→ `territoryBonus -1` |
| Firestore | `camera_jack`, `spectral_territory` | `facility_sabotage`, `camera_shutdown` |

- 脱落数だけでは告発有効施設は増えない（解禁で1、陣取りで±）
- 本隊の告発・逃走はそのまま；第二ゲームは情報戦レイヤー

## 人狼

| 種類 | 仕様 |
|------|------|
| **任意切替** | ボタン表示は状態に応じ **鬼化 / 人化**。切替可能まで = **自動切替間隔 ÷ 3**（前回切替から） |
| **自動切替** | **前回切替（自動・任意どちらでも）から** `min(900秒, 試合時間÷2)` 経過で **鬼⇄人を自動トグル**（通知なし） |
| **自動直後の待ち** | 切替可能まで + 切替可能まで÷2（通常よりやや長い） |
| **HUD** | **「自動切替まで」**と**「切替可能まで」**を別チップ表示 |

### ロール vs 陣営

| 概念 | 意味 |
|------|------|
| **人ロール / 鬼ロール** | 周囲から見た現在の立場（人狼は鬼化中=鬼ロール、人の姿=人ロール） |
| **人陣営 / 鬼陣営** | 勝敗・脱落後第二ゲームの所属（状態と独立） |

**人狼の陣営**（自分以外の生存者で数える）:

- 人ロール数 ≤ 鬼ロール数 → **人陣営**（鬼化中は捕獲可・感染も可）
- 人ロール数 > 鬼ロール数 → **鬼陣営**（鬼化中は感染等のみ・**捕獲不可**）

- 人狼は **告発不可**
- 鬼同士（本鬼＋鬼化人狼）は **感染・捕獲しない**
- 脱落時の第二ゲームは **脱落時点の陣営**で分岐（人陣営→残響体、鬼陣営→復讐の鬼影）。**第二ゲーム中は陣営固定**（その後の人数変動で切り替わらない）

`lib/game/werewolf_faction_logic.dart` · `lib/game/werewolf_forced_schedule.dart`

### 鬼の遅延軌跡

- ライブ位置は共有しない。遅延帯は **`MatchDurationScaling.oniTrail`**（試合の約22〜38%前）
- 序盤は **試合開始付近の鬼位置ピン**（試合の約15%経過まで）

## 人狼脱落と第二ゲーム

**分岐:** 脱落時点の **陣営**（`forEliminatedFaction`）。人陣営→残響体、鬼陣営→復讐の鬼影。

**3人戦の例:** 人狼は人陣営になりやすい → 序盤にエリア外脱落しても **残響体**（鬼影サポートの抜け道を塞ぐ）。

**5〜6人戦の例:** 人狼は鬼陣営になりやすい → 鬼化しても直接捕獲できず、本鬼に人を殺させる攪乱が必要。脱落が進むと陣営が反転し終盤は人側に寄せられる。

## プレイエリア（編集・保存・試合反映）

### 正しい流れ（2段階）

```
[エリア編集] →「エリアを保存」→ スロット一覧に追加（この時点では試合に未反映）
                    ↓
         ホストがスロットを「選択」→「適用」→ _playArea が更新 + 端末に play_area_v1 保存
                    ↓
              「試合開始」→ オンラインは Firestore に playArea 同梱で全員同期
```

| 操作 | `_playArea`（試合で使う形） | 他プレイヤー |
|------|---------------------------|--------------|
| エリアを保存（スロット） | **変わらない** | 見えない |
| ホスト「適用」 | **更新** | オンライン時は `lobby_play_area` イベントで **全員の地図に反映**（sessionKey=0） |
| 試合開始（オンライン） | ホストの形が **全員に確定** | `SharedMatchSnapshot.playArea` で一致 |
| GeoJSON 取込 | **即 `_playArea` 更新** + `play_area_v1` 保存 | ホスト・オンライン時は `lobby_play_area` も配信（`slotId: geojson_import`） |

### 試合開始時の自動補正

- ホストが開始直前に `_alignPlayAreaToCurrentPositionIfFar()` を実行
- **円エリアのみ**: 現在地がエリア外かつ **80m 以上** はみ出している → 中心を現在地に移動（半径は維持）
- **多角形**: 自動補正 **なし**（遠い場所で開始するとエリア外扱いになりうる）

### 起こりうる不具合・混乱（コード上の仕様）

| 症状 | 原因 |
|------|------|
| 保存したのに試合で昔のエリア | **保存≠適用**。ホストが「適用」していない |
| 非ホストの地図がホストと違う | ホストが「適用」前、またはオフライン／イベント未到達。適用後は `lobby_play_area` で一致する想定 |
| 円を別の街で保存したが開始できる | 開始時 **80m 以上** なら円の中心だけ現在地に寄せる（ホストのみ） |
| 多角形を離れた場所で開始 | 自動寄せなし → **エリア外暴露・90秒脱落** が早く入る |
| GeoJSON とスロットで挙動が違う | 取込は即反映、スロットは適用が必要 |
