# ONI PIN Diagram Design

## 1. このファイルの目的

このファイルは、ONI PINの遊び方・チュートリアル・作戦マニュアルに入れる図解の設計書である。

ONI PINは文章だけでは理解しにくい。

理由は、ゲームの核が

```text
位置を見ること
```

ではなく

```text
位置を推測すること
```

にあるためである。

したがって、図解は飾りではない。

図解は、プレイヤーがゲーム構造を理解するための主要UIである。

---

# 2. 図解の基本方針

## DIA-001: 1図1テーマ

1つの図で複数のルールを説明しすぎない。

悪い例:

```text
情報戦、パニック、告発、第二ゲームを1枚にまとめる
```

良い例:

```text
情報戦だけ
鬼との距離だけ
エリア外だけ
告発だけ
```

## DIA-002: 5秒で分かる図にする

プレイヤーが試合中に開いても理解できるようにする。

細かい説明は図の下に置く。

## DIA-003: 数字より構造を見せる

図では、秒数やメートルよりも

```text
流れ
関係
危険度
分岐
```

を優先する。

数値は詳細ルールに置く。

## DIA-004: 現在地ではなく「痕跡」を強調する

ONI PINの最重要概念は、

```text
地図の点 = 現在地とは限らない
```

である。

図では必ずこの感覚を伝える。

## DIA-005: 実装は段階的でよい

最初は簡易Widget、絵文字、アイコン、線、カードで実装してよい。

将来SVGやCustomPainterに置き換えられるように、構造を分ける。

---

# 3. 共通DiagramCard仕様

推奨Widget名:

```text
GuideDiagramCard
```

基本構造:

```text
GuideDiagramCard
├─ title
├─ subtitle
├─ visual
├─ caption
└─ optional legend
```

例:

```text
title: 情報は「現在地」ではなく「手がかり」
subtitle: ライブ位置は基本見えません
visual: 図本体
caption: 点が出ても、今そこにいるとは限りません。
legend: 📍名前付き / ？匿名
```

# 5. DIA-CORE-001: ONI PINは情報戦

## 目的

ゲームの最初に、

```text
これは単なるGPS鬼ごっこではなく、情報を読むゲーム
```

だと伝える。

## 表示場所

* ガイドブック冒頭
* はじめにセクション
* チュートリアル開始前の説明にも流用可

## 簡易図

```text
相手の現在地
    × 見えない

        ↓

見えるのは手がかり

📍 名前付き暴露
？ 匿名痕跡
📷 監視カメラ
📡 情報屋

        ↓

情報を読んで
逃げる・追う・告発する
```

## UI構成案

```text
[大きな×付きの位置ピン]
↓
[4つの小カード]
  📍 名前付き暴露
  ？ 匿名痕跡
  📷 カメラ
  📡 情報屋
↓
[判断カード]
  逃げる / 追う / 告発
```

## キャプション

```text
ONI PINでは、位置そのものではなく、位置の手がかりを読みます。
```

# 6. DIA-WIN-001: 勝敗条件

## 目的

人陣営と鬼陣営の勝ち方を一目で見せる。

## 表示場所

* 勝ち方セクション

## 簡易図

```text
人陣営の勝ち
├─ 時間切れまで生存
├─ 本鬼を告発
└─ 本鬼が0人

鬼陣営の勝ち
└─ 人側生存者を0人
```

## UI構成案

左右2カラム。

```text
┌────────────┐  ┌────────────┐
│ 人陣営      │  │ 鬼陣営      │
│ 🏃 生存     │  │ 😈 捕獲     │
│ 🏛 告発     │  │ 全員脱落    │
│ 👹 本鬼0    │  │             │
└────────────┘  └────────────┘
```

## 注意

必ず以下を補足する。

```text
人狼が鬼陣営に残っていても、本鬼が0人なら人陣営勝利。
```

# 8. DIA-INFO-002: 名前付き暴露と匿名痕跡

## 目的

2種類の情報の違いを一目で理解させる。

## 表示場所

* 情報戦セクション
* チュートリアル内

## 簡易図

```text
📍 名前付き暴露
「ミソラがここにいた」
強いが、今いるとは限らない

？ 匿名痕跡
「誰かがここにいた」
弱いが、つなぐと読める
```

## UI構成案

上下または左右2カード。

```text
┌──────────────┐
│ 📍 名前付き暴露 │
│ ○○がここにいた │
│ 名前あり        │
└──────────────┘

┌──────────────┐
│ ？ 匿名痕跡    │
│ 誰かがここにいた │
│ 名前なし        │
└──────────────┘
```

## キャプション

```text
名前付きは強い情報。匿名は推理の材料です。
```

# 9. DIA-INFO-003: 情報の強さ

## 目的

情報には強弱があることを示す。

ただし、匿名痕跡も複数集まると強くなることを示す。

## 表示場所

* 情報戦セクション詳細
* 上級者向けTips

## 簡易図

```text
強い
↑
📍 名前付き暴露
📡 情報屋ヒント
📷 監視カメラ
？ 匿名痕跡
↓
弱い

ただし…
？ + ？ + ？ = 移動方向が見える
```

## UI構成案

縦の強弱バー。

```text
強い情報
┃ 📍 名前付き暴露
┃ 📡 情報屋
┃ 📷 カメラ反応
┃ ？ 匿名痕跡
弱い情報
```

下に補足カード:

```text
匿名痕跡は1つだと弱い。
でも複数つながると、逃げた方向が見えてきます。
```

## 注意

「匿名痕跡 = 弱いから無視」と誤解されないようにする。

```text

      ↓
   ？ 10:02
          ↓
       📷 10:03
              ↓
           📍 ○○ 10:04

移動方向を予測
```

## UI構成案

地図風の点列。

```text
┌────────────────┐
│ ?               │
│    ?            │
│       📷        │
│          📍○○   │
└────────────────┘

痕跡をつなぐと、移動方向が見えてきます。
```

## キャプション

```text
点を1つずつ見るのではなく、時間と方向をつないで読みます。
```

# 11. DIA-COMBAT-001: 鬼との距離と危険度

## 目的

鬼に近づくほど危険が増す構造を示す。

## 表示場所

* 鬼との戦いセクション冒頭
* 逃走者チュートリアル
* 本鬼チュートリアル

## 簡易図

```text
        😈 鬼

    🔴 捕獲
  🟠 拘束
🟡 パニック
🟢 安全

        🧍
```

## UI構成案

同心円または縦レイヤー。

同心円が難しければ、縦に危険度を並べる。

```text
┌──────────────┐
│ 😈 鬼          │
│ 🔴 捕獲        │
│ 🟠 拘束        │
│ 🟡 パニック    │
│ 🟢 安全        │
│ 🧍 逃走者      │
└──────────────┘
```

## キャプション

```text
パニックは警告、拘束はピンチ、捕獲は脱落です。
```

# 13. DIA-COMBAT-003: 拘束から捕獲まで

## 目的

接触・拘束・捕獲の流れを理解させる。

## 表示場所

* 鬼との戦い詳細
* 本鬼チュートリアル

## 簡易図

```text
鬼が近い
↓
接触
↓
拘束
↓
拘束円から逃げる
├─ 逃げ切る → セーフ
└─ 戻れない → 暴露 → 捕獲
```

## UI構成案

分岐フロー図。

```text
[接触]
   ↓
[拘束]
   ↓
[拘束円の外へ]
   ├─ 戻れる → セーフ
   └─ 戻れない → 📍暴露 → 捕獲
```

## キャプション

```text
拘束は即脱落ではありません。逃げ切る余地があります。
```

# 14. DIA-AREA-001: エリア外の危険段階

## 目的

エリア外が即死ではなく段階的に危険になることを示す。

## 表示場所

* エリア外セクション

## 簡易図

```text
エリア外に出る
↓
少しの猶予
↓
📍 名前付き暴露
↓
📍 再暴露
↓
💀 脱落
```

## UI構成案

タイムライン。

```text
┌──────────────┐
│ エリア外      │
└─────↓────────┘
  猶予
    ↓
  📍 暴露
    ↓
  📍 再暴露
    ↓
  💀 脱落
```

## キャプション

```text
エリア外は逃げ道にもなりますが、長くいるほど位置がバレます。
```

## 注意

鬼についても補足する。

```text
本鬼もエリア外が続くと脱落します。
```

# 16. DIA-ACCUSE-001: 告発の流れ

## 目的

告発が「本鬼を推理して当てる」仕組みであることを示す。

## 表示場所

* 告発セクション

## 簡易図

```text
？ 匿名痕跡
📍 名前付き暴露
📷 カメラ

        ↓

本鬼を推理

        ↓

🏛 告発施設

        ↓

正解なら勝利
```

## UI構成案

```text
[情報を集める]
  ?  📍  📷
       ↓
[推理する]
  🤔 本鬼は誰？
       ↓
[告発施設へ]
  🏛
       ↓
[結果]
  正解 → 🏆
  失敗 → 脱落リスク
```

## キャプション

```text
告発は勘ではなく、痕跡と暴露を読んで使う逆転手段です。
```

class GuideFlow extends StatelessWidget {
  const GuideFlow({
    super.key,
    required this.items,
  });

  final List<String> items;

  @override
  Widget build(BuildContext context) {
    return Column(
      children: [
        for (var i = 0; i < items.length; i++) ...[
          Text(items[i]),
          if (i != items.length - 1)
            const Icon(Icons.keyboard_arrow_down),
        ],
      ],
    );
  }
}
