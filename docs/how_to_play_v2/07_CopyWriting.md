# ONI PIN CopyWriting Guide

## 1. このファイルの目的

このファイルは、ONI PIN内で使うプレイヤー向け文言を統一するためのコピーライティング指針である。

対象は以下。

* 遊び方ガイド
* 役職開示ダイアログ
* チュートリアル文言
* HUD通知
* イベントログ
* スキル説明
* 脱落後表示
* 告発表示
* エラー・注意表示

目的は、文章をかっこよくすることではない。

目的は、プレイヤーが迷わず理解できるようにすることである。

# 2. 基本トーン

## COPY-001: 短く、強く、分かりやすく

試合中に読む文は短くする。

悪い例:

```text
あなたは鬼のパニック圏内に一定時間滞在したため、匿名痕跡が発生する状態になりました。
```

良い例:

```text
パニック発生
匿名痕跡が出ます。
```

## COPY-002: 仕様語ではなく体験語を使う

内部処理ではなく、プレイヤーの体験に沿って書く。

悪い例:

```text
infection状態になりました。
```

良い例:

```text
パニックになりました。
```

## COPY-003: 数値より意味を先に出す

数値は必要なときだけ出す。

悪い例:

```text
7秒ごとに匿名痕跡が発生します。
```

良い例:

```text
しばらく匿名痕跡が出続けます。
```

詳細欄では数値を書いてよい。

## COPY-004: 不安を煽りすぎない

危険は伝えるが、何をすればよいかも一緒に示す。

悪い例:

```text
危険です。捕獲されます。
```

良い例:

```text
鬼が近くにいます。
距離を取りましょう。
```

## COPY-005: 「今そこにいる」と断定しない

暴露や痕跡は、現在地とは限らない。

悪い例:

```text
○○はここにいます。
```

良い例:

```text
○○がここで暴露されました。
```

または

```text
○○がこの付近にいました。
```

# 3. 禁止・非推奨ワード

## 必ず避ける

| NG                | 使う表現   |
| ----------------- | ------ |
| 感染                | パニック   |
| infection         | パニック   |
| anonymous reveal  | 匿名痕跡   |
| identified reveal | 名前付き暴露 |
| hunter            | 本鬼     |
| runner            | 逃走者    |
| lockZone          | 拘束円    |
| captureZone       | 捕獲結界   |
| touchLock         | 接触拘束   |
| echo body         | 残響体    |
| revenge shadow    | 復讐の鬼影  |

## 注意して使う語

### 「位置」

「現在地」と誤解されやすい。

なるべく以下を使う。

* 位置情報
* 暴露地点
* 痕跡
* 手がかり
* この付近

### 「見つかった」

強すぎる場合がある。

名前付き暴露では使ってよい。

匿名痕跡では避ける。

### 「捕まった」

捕獲時のみ使う。

パニックや拘束では使わない。

# 5. 役職開示ダイアログ文案

## 5.1 逃走者

タイトル:

```text
あなたは逃走者です
```

本文:

```text
鬼から逃げながら、痕跡と暴露を読んで本鬼を見抜きましょう。

相手のライブ位置は基本見えません。
情報を残しすぎない動きが大切です。
```

ボタン:

```text
チュートリアル
作戦マニュアル
試合へ
```

## 5.2 本鬼

タイトル:

```text
あなたは本鬼です
```

本文:

```text
逃走者を追い詰め、捕獲しましょう。

逃走者のライブ位置は基本見えません。
匿名痕跡や名前付き暴露を読んで追跡します。
```

ボタン:

```text
チュートリアル
作戦マニュアル
試合へ
```

## 5.3 人狼

タイトル:

```text
あなたは人狼です
```

本文:

```text
人狼は、鬼化で場を乱せる特殊役職です。

ただし、本鬼ではありません。
告発対象や勝敗条件では、本鬼とは別に扱われます。
```

ボタン:

```text
チュートリアル
役職を詳しく見る
試合へ
```

## 5.4 アナリスト

タイトル:

```text
あなたはアナリストです
```

本文:

```text
匿名痕跡を読み解くことに長けた逃走者です。

痕跡の時間帯や発生源を読み、鬼の動きを推理しましょう。
```

## 5.5 ハッカー

タイトル:

```text
あなたはハッカーです
```

本文:

```text
情報屋から、より精密な手がかりを得やすい逃走者です。

方角や距離帯を読み、味方の判断を助けましょう。
```

# 17. よくある誤解への返答文

## 地図の点は現在地ですか？

```text
多くの場合、現在地ではありません。
一度バレた場所や、誰かがいた痕跡です。
```

## パニックになったら脱落ですか？

```text
脱落ではありません。
ただし、匿名痕跡が出るため追われやすくなります。
```

## 拘束されたら終わりですか？

```text
すぐには終わりません。
拘束円から逃げ切れば助かります。
```

## 人狼を告発すれば勝ちですか？

```text
いいえ。
告発で当てるべきなのは本鬼です。
人狼は本鬼ではありません。
```

## 鬼がエリア外に出たら即勝ちですか？

```text
いいえ。
鬼もエリア外が続くと脱落します。
本鬼が0人になると人陣営の勝ちです。
```

## 脱落したら何もできませんか？

```text
いいえ。
第二ゲームで味方を助けたり、相手を妨害したりできます。
```

# 18. 文章チェックリスト

実装前に、すべての文言について以下を確認する。

* 「感染」と表示していない
* パニックを脱落扱いしていない
* 匿名痕跡に名前を付けていない
* 名前付き暴露を現在地と断定していない
* 人狼を本鬼として扱っていない
* 告発できる人を曖昧にしていない
* 鬼エリア外を即人勝利として書いていない
* 脱落後を観戦だけとして書いていない
* 数値をトップレベルに出しすぎていない
* 仕様書にない効果を追加していない

# 19. 実装時の推奨

文言は可能なら定数化する。

例:

```dart
class MatchCopy {
  static const panicTitle = 'パニック発生';
  static const panicBody = '匿名痕跡が出ます。離れて立て直しましょう。';
}
```

既に `MatchUiTerms` のような用語管理がある場合は、それを優先し、追加する形にする。

同じ意味の文言を複数箇所に直書きしない。

このコピーガイドに従った実装では、以下が満たされること。

* ガイドブックとHUD通知の用語が一致している
