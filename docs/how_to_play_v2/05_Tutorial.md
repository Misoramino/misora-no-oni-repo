# ONI PIN Role Tutorial Design

## 1. このファイルの目的

このファイルは、役職開示ダイアログから入れる約1分の役職別チュートリアルを改善するための設計書である。

ガイドブックは「困ったときに調べる作戦マニュアル」。

チュートリアルは「実際に体験して覚えるミニゲーム」。

この2つは役割が違う。

チュートリアルでは、正確な仕様をすべて説明しない。

プレイヤーがその役職で何を考えればよいかを、短い体験として伝える。

---

# 2. チュートリアル全体方針

## TUT-001: 1分以内で終わる

各チュートリアルは約1分を目標にする。

長くても90秒を超えない。

## TUT-002: 説明ではなく体験にする

長文説明を出さない。

プレイヤーに実際に移動・確認・タップ・回避・追跡をさせる。

## TUT-003: 役職らしい考え方を教える

操作方法だけではなく、その役職の勝ち方の考え方を伝える。

例:

* 逃走者: 逃げるだけでなく、痕跡を残しすぎない
* 本鬼: 速さだけでなく、情報を読む
* 人狼: 本鬼ではないが、場を乱せる
* 脱落後: 終わりではなく、第二ゲームに入る

## TUT-004: 数値は出しすぎない

チュートリアル中は秒数や距離を最小限にする。

詳細はガイドブックに誘導する。

## TUT-005: チュートリアル後に関連ページへ誘導する

終了画面に以下を出す。

```text
もっと詳しく見る
・情報戦
・鬼との戦い
・役職
```

役職に応じてリンク先を変える。

# 4. 逃走者チュートリアル

## 目的

逃走者には、次を体験させる。

* ライブ位置は見えない
* 匿名痕跡が出る
* 鬼に近づくとパニックになる
* パニックは脱落ではない
* 痕跡を残しすぎないことが重要
* 告発施設の存在を知る

## コンセプト

```text
逃げるだけでなく、情報を残さず逃げる。
```

## 流れ

### Step 1: 地図を見る

表示:

```text
ミッション 1/5
地図を確認しましょう。
```

説明:

```text
相手のライブ位置は基本見えません。
地図に出るのは、痕跡や暴露などの手がかりです。
```

操作:

```text
プレイヤーに地図を少し動かさせる、または現在地を確認させる。
```

成功条件:

```text
一定時間経過、または地図操作。
```

### Step 2: 匿名痕跡を見る

表示:

```text
ミッション 2/5
匿名痕跡を確認しましょう。
```

演出:

```text
近くに「？」付きの痕跡を1つ表示する。
```

説明:

```text
これは匿名痕跡です。
誰の位置かは分かりませんが、「誰かがここにいた」手がかりになります。
```

成功条件:

```text
痕跡をタップ、または近くまで移動。
```

### Step 3: 鬼に近づきすぎる

表示:

```text
ミッション 3/5
鬼の近くにいると危険です。
```

演出:

```text
チュートリアル用の鬼または危険エリアを表示する。
プレイヤーが近づくと、パニック警告を出す。
```

説明:

```text
鬼に近づきすぎると、パニックになります。
パニックは脱落ではありませんが、匿名痕跡が出て追われやすくなります。
```

成功条件:

```text
パニック演出を見る。
```

### Step 4: 痕跡を切る

表示:

```text
ミッション 4/5
鬼から距離を取りましょう。
```

説明:

```text
パニック中は痕跡が残ります。
大きく移動して、鬼に読まれにくい位置へ逃げましょう。
```

操作:

```text
指定された安全方向へ移動、または仮想移動操作。
```

成功条件:

```text
危険エリアから離れる。
```

### Step 5: 告発施設を見る

表示:

```text
ミッション 5/5
本鬼を見抜いたら、告発施設へ。
```

演出:

```text
告発施設アイコンを表示する。
```

説明:

```text
3人以上の試合では、後半に告発施設が使えます。
本鬼を当てると、人陣営の大きな勝ち筋になります。
```

成功条件:

```text
施設をタップ、または説明を確認。
```

## 終了文

```text
逃走者チュートリアル完了

逃走者は、ただ逃げるだけではありません。
痕跡を残しすぎず、情報を読み、本鬼を見抜くことが大切です。
```

関連ガイド:

```text
情報戦
鬼との戦い
告発
```

# 5. 本鬼チュートリアル

## 目的

本鬼には、次を体験させる。

* 逃走者のライブ位置は見えない
* 匿名痕跡と名前付き暴露を読む
* パニック痕跡で追跡する
* 近づいて拘束する
* 捕獲する
* 告発施設を守れる

## コンセプト

```text
鬼は足の速さだけでなく、情報を読む力で追い詰める。
```

## 流れ

### Step 1: 逃走者は見えない

表示:

```text
ミッション 1/5
逃走者のライブ位置は見えません。
```

説明:

```text
本番では、逃走者の現在地が常に見えるわけではありません。
痕跡や暴露から逃走者の動きを読みます。
```

演出:

```text
地図上に逃走者ピンを出さず、痕跡だけ表示する。
```

### Step 2: 匿名痕跡を読む

表示:

```text
ミッション 2/5
匿名痕跡を追いましょう。
```

演出:

```text
匿名痕跡を2〜3個、移動方向が分かるように表示する。
```

説明:

```text
匿名痕跡は、誰かがいた手がかりです。
複数つなげると、逃げた方向が見えてきます。
```

成功条件:

```text
最後の痕跡付近へ移動、または痕跡を順番にタップ。
```

### Step 3: 名前付き暴露を見る

表示:

```text
ミッション 3/5
名前付き暴露を確認しましょう。
```

演出:

```text
「○○がここにいた」という名前付き暴露を表示する。
```

説明:

```text
名前付き暴露は強い情報です。
ただし、今もそこにいるとは限りません。
移動先を予測して追いましょう。
```

成功条件:

```text
暴露位置をタップ。
```

### Step 4: パニック痕跡を追う

表示:

```text
ミッション 4/5
パニック痕跡を追いましょう。
```

演出:

```text
逃走者がパニックになった想定で、匿名痕跡を連続表示する。
```

説明:

```text
逃走者が鬼の近くでパニックになると、匿名痕跡が出ます。
この痕跡は追跡の大きな手がかりになります。
```

成功条件:

```text
痕跡の先へ移動。
```

### Step 5: 拘束・捕獲

表示:

```text
ミッション 5/5
近づいて捕獲しましょう。
```

演出:

```text
チュートリアル用の逃走者を表示。
鬼が近づくと拘束演出。
さらに近づくと捕獲演出。
```

説明:

```text
逃走者に近づき続けると拘束できます。
至近距離まで追い詰めると捕獲です。
```

成功条件:

```text
捕獲演出を見る。
```

## 補足演出: 告発施設を守る

チュートリアルが長くなりすぎる場合、終了画面に短く出す。

```text
本鬼は、告発施設の近くにいることで、その施設での告発を封じることがあります。
終盤は施設を守る動きも重要です。
```

## 終了文

```text
本鬼チュートリアル完了

本鬼は、逃走者のライブ位置を追うのではなく、痕跡と暴露から動きを読む役職です。
情報をつなげて、逃走者を追い詰めましょう。
```

関連ガイド:

```text
情報戦
鬼との戦い
告発
```

# 6. 人狼チュートリアル

## 目的

人狼には、次を体験させる。

* 人狼は本鬼ではない
* 鬼化できる
* 状況によって立場が変わる
* 告発できない
* 本鬼0人で人陣営勝利になる
* 詳細はガイドブックで確認する

## コンセプト

```text
人狼は、場を乱す特殊役職。本鬼ではない。
```

## 注意

人狼の仕様は複雑である。

チュートリアルで細かい陣営判定を説明しすぎない。

最重要なのは、

```text
鬼化できるが、本鬼ではない
```

を理解させること。

## 流れ

### Step 1: 自分は人狼

表示:

```text
あなたは人狼です。
```

説明:

```text
人狼は、状況によって鬼のように動ける特殊役職です。

ただし、本鬼ではありません。
```

成功条件:

```text
確認ボタン。
```

### Step 2: 鬼化する

表示:

```text
ミッション 1/3
鬼化してみましょう。
```

操作:

```text
鬼化ボタンを押す。
```

説明:

```text
鬼化中は、鬼のように相手を追ったり、拘束したりできます。
```

成功条件:

```text
鬼化状態になる。
```

### Step 3: 本鬼ではないことを確認

表示:

```text
ミッション 2/3
人狼は本鬼ではありません。
```

説明:

```text
人陣営が告発で当てるべきなのは本鬼です。

人狼は鬼のように見えることがありますが、本鬼とは別の役職です。
```

演出:

```text
本鬼アイコンと人狼アイコンを並べて表示し、別物であることを示す。
```

### Step 4: 告発不可を確認

表示:

```text
ミッション 3/3
人狼は告発できません。
```

説明:

```text
告発できるのは、生存中の逃走者です。

人狼は告発できません。
```

成功条件:

```text
説明確認。
```

## 終了文

```text
人狼チュートリアル完了

人狼は、本鬼ではありません。
鬼化で場を乱しながら、状況に応じて立ち回る特殊役職です。

細かい立場の変化は、作戦マニュアルの「役職」を確認してください。
```

関連ガイド:

```text
役職
スキル
告発
```

# 7. 脱落後チュートリアル

## 目的

脱落後に、プレイヤーが「もう何もできない」と思わないようにする。

第二ゲームでできることを短く体験させる。

## 表示タイミング

初めて脱落したとき。

または役職別チュートリアル内のオプションとして表示。

## コンセプト

```text
脱落は終わりではない。第二ゲームの始まり。
```

# 8. チュートリアル文体ルール

## 短く言う

チュートリアル中の説明は短くする。

悪い例:

```text
パニック状態では約22秒間、7秒ごとに匿名痕跡が発生し、偽位置スキル中はおとり座標に補正されることがあります。
```

良い例:

```text
パニック中は匿名痕跡が出ます。
脱落ではありませんが、追われやすくなります。
```

## 数値は出しすぎない

チュートリアル中に数値を出す場合は、プレイヤー行動に必要なものだけにする。

詳細な秒数はガイドブックへ。

## 役職の目的を必ず言う

各チュートリアルの最後に、その役職で一番大切な考え方を表示する。

例:

```text
逃走者は、情報を残さず逃げる。
```

```text
本鬼は、痕跡を読んで追う。
```

```text
人狼は、本鬼ではない特殊役職。
```

# 10. 実装メモ

## 既存チュートリアルがある場合

既存のミニゲーム構造を優先してよい。

ただし、説明文と体験順はこの設計に合わせる。

## 実装負荷を下げる場合

最初の実装では、完全な移動シミュレーションでなくてもよい。

以下でもよい。

* 地図上に仮想マーカーを表示
* タップで進む
* 簡易アニメーション
* サンプル痕跡を表示
* スキルボタンの疑似操作

## 優先順位

最優先:

* 逃走者チュートリアル
* 本鬼チュートリアル
* 人狼チュートリアル

次点:

* 残響体チュートリアル
* 復讐の鬼影チュートリアル

後回し可:

* 詳細な演出
* 複雑な地図移動
* 実距離に基づくチュートリアル判定

---

# 11. 実装時の禁止事項

* チュートリアル中に長文仕様説明を出さない
* パニックを「感染」と表示しない
* 人狼を本鬼として説明しない
* 告発できる人を曖昧にしない
* 脱落後を観戦だけとして説明しない
* 数値説明をチュートリアルの中心にしない
* 既存ゲームロジックをチュートリアルのために変更しない

---

# 12. 完了条件

チュートリアル実装後、次を満たしていること。

* 逃走者が「痕跡を残しすぎない」意識を持てる
* 本鬼が「痕跡を読んで追う」意識を持てる
* 人狼が「本鬼ではない」と理解できる
* 脱落後も試合に関与できると理解できる
* チュートリアル後に関連ガイドへ移動できる
* ガイドブックと用語が一致している
