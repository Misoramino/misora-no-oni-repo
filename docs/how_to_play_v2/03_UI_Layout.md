# ONI PIN HowToPlay Ver2 UI Layout

## 1. このファイルの目的

このファイルは、ONI PINの遊び方ガイドをどのようなUIで表示するかを定義する。

目的は、既存の長文説明を、プレイヤーが試合中でも読みやすい「作戦マニュアル型UI」に再構成することである。

---

# 2. 基本方針

## UI-001: 全部読ませない

プレイヤーにすべてのルールを一度に読ませない。

トップでは章だけを見せる。

各章では概要を見せる。

詳細は必要なときだけ開ける。

## UI-002: カード型にする

章の中身は、短いカードの集合にする。

1カードにつき1テーマ。

1カードで複数の概念を説明しない。

## UI-003: 折りたたみを使う

秒数・距離・クールダウン・例外は、原則として折りたたみに入れる。

初心者向けの説明と上級者向けの正確な仕様を分ける。

## UI-004: 図解を入れる

文章だけにしない。

ONI PINは空間・距離・痕跡・追跡のゲームなので、図解が必要。

図解は最初はASCII・簡易Widgetでよい。

後からSVGやCustomPainterに置き換えてよい。

## UI-005: 試合中に読める長さにする

1カードの概要文は、原則3〜5行以内にする。

詳細欄は長くなってもよいが、最初から展開しない。

# 3. 全体画面構成

推奨画面名:

* `HowToPlayGuideScreen`
* または既存の `HowToPlayContent` を拡張

基本構造:

```text
Scaffold
└─ AppBar
   └─ title: 作戦マニュアル
└─ Body
   └─ ListView
      ├─ HeaderCard
      ├─ SectionIndex
      ├─ GuideSection: はじめに
      ├─ GuideSection: 勝ち方
      ├─ GuideSection: 情報戦
      ├─ GuideSection: 鬼との戦い
      ├─ GuideSection: エリア外
      ├─ GuideSection: マップ施設
      ├─ GuideSection: 告発
      ├─ GuideSection: 役職
      ├─ GuideSection: スキル
      ├─ GuideSection: 第二ゲーム
      ├─ GuideSection: オンライン・記録
      └─ GuideSection: 詳細ルール
```

# 4. トップヘッダー

## 目的

最初に、ONI PINが何のゲームかを1画面で伝える。

## 表示内容

タイトル:

```text
ONI PIN 作戦マニュアル
```

サブタイトル:

```text
位置は見えない。痕跡を読み、鬼を出し抜け。
```

説明:

```text
ONI PINは、屋外GPSタッグに人狼と情報戦を組み合わせたゲームです。
相手のライブ位置は基本見えません。
名前付き暴露や匿名痕跡を読みながら、逃げる・追う・告発する判断を行います。
```

補足ラベル:

```text
困ったときに必要な章だけ開いてください。
```

# 6. GuideSectionの構造

各章は以下の構造にする。

```text
GuideSection
├─ SectionHeader
│  ├─ icon
│  ├─ title
│  └─ oneLineSummary
├─ DiagramCard
├─ GuideCard
├─ GuideCard
└─ DetailExpansion
```

## SectionHeader

例:

```text
情報戦
ライブ位置は見えません。痕跡から相手を読みます。
```

## DiagramCard

その章の最重要概念を図で示す。

## GuideCard

短い説明カード。

## DetailExpansion

正確な数値・例外・上級者向け仕様。

---

# 7. GuideCardの構造

```text
GuideCard
├─ icon
├─ title
├─ oneLine
├─ body
├─ optional bullets
└─ optional detail expansion
```

## 例

```text
名前付き暴露

「○○がここにいた」と分かる強い情報です。

本文:
エリア外、告発失敗、情報屋などで発生します。
名前は出ますが、現在地そのものではありません。

詳細:
発生源一覧...
```


# 9. 詳細折りたたみ

詳細欄の見出し例:

* 正確な仕様
* 発生する条件
* よくある誤解
* 数値の目安
* 上級者向け

## 詳細欄に入れるもの

* 秒数
* 距離
* クールダウン
* 上限回数
* 例外
* ホスト設定差分
* オンライン同期の細部

## 詳細欄に入れないもの

* その章の最重要概念
* 初心者が知らないと遊べないこと
* 勝敗に直結する基本条件

# 10. 見た目の方向性

## 世界観

ガイドは「作戦マニュアル」として見せる。

ただし、読みづらい専門用語にはしない。

## トーン

* 緊張感
* 情報戦
* 読みやすさ
* ゲーム内マニュアル感

## 避けるもの

* 仕様書のような長文羅列
* コード用語
* 内部実装名
* 数値だけの説明
* 例外から先に入る説明

# 11. 色・アイコンの提案

既存デザインに合わせること。

新規に指定する場合は以下を目安にする。

| 概念     | アイコン案            | 色の方向          |
| ------ | ---------------- | ------------- |
| はじめに   | 📖 / info        | neutral       |
| 勝ち方    | 🏁 / flag        | accent        |
| 情報戦    | 📡 / radar       | blue/cyan     |
| 名前付き暴露 | 📍 / person_pin  | red/orange    |
| 匿名痕跡   | ? / blur_on      | purple/gray   |
| 鬼との戦い  | 😈 / warning     | red           |
| パニック   | ⚠ / crisis_alert | yellow/orange |
| 捕獲     | ⛓ / lock         | red           |
| エリア外   | 🧭 / boundary    | orange        |
| 施設     | 🏢 / map         | green         |
| 告発     | ⚖ / report       | violet        |
| 役職     | 🎭 / groups      | indigo        |
| スキル    | ✨ / bolt         | amber         |
| 第二ゲーム  | 👻 / replay      | gray/purple   |
| オンライン  | ☁ / sync         | blue          |

絵文字をそのまま使うか、Material Iconsへ置き換えるかは既存UIに合わせる。

# 12. 関連リンク

各章の末尾に、関連章への導線を置く。

例:

## 情報戦の末尾

```text
関連:
・鬼との戦い
・マップ施設
・詳細ルール
```

## 鬼との戦いの末尾

```text
関連:
・情報戦
・スキル
・詳細ルール
```

## 告発の末尾

```text
関連:
・情報戦
・役職
・第二ゲーム
```

これにより、長い説明を一本道で読ませず、必要な場所へ移動できる。

# 13. チュートリアルとの導線

役職開示ダイアログに、既存通りチュートリアル導線を置く。

ガイドブック側にも、可能なら以下のような導線を置く。

```text
この内容を体験する:
逃走者チュートリアル
鬼チュートリアル
人狼チュートリアル
```

チュートリアル終了時には、関連ガイドを開ける導線を置く。

例:

```text
もっと詳しく:
情報戦を見る
鬼との戦いを見る
```

# 14. 詳細ルールページ

詳細ルールページは、仕様表として使う。

他のページでは「約」「目安」を使ってよい。

詳細ルールでは、実装準拠の数値をまとめる。

ただし、ここでもコード名はできるだけ出さない。

# 16. 実装時の禁止事項

以下は禁止。

* 文章を1つの巨大なTextにまとめる
* 詳細仕様を全部トップに表示する
* パニックを「感染」と表示する
* 人狼を本鬼と同じ扱いで説明する
* 鬼エリア外を即人勝利と書く
* 匿名痕跡を名前付き暴露のように説明する
* 告発を誰でもできるように書く
* 脱落後を観戦だけのように説明する
* 仕様書にない新ルールを追加する

---

# 17. 完成後の確認項目

実装後、次を確認する。

* トップから重要章にすぐ移動できる
* 情報戦ページだけで名前付き暴露と匿名痕跡の違いが分かる
* 鬼との戦いページだけでパニック・拘束・捕獲の違いが分かる
* エリア外ページだけで暴露→再暴露→脱落の流れが分かる
* 告発ページだけで告発できる人・対象・施設封鎖が分かる
* 人狼ページで「本鬼ではない」と分かる
* 第二ゲームページで脱落後も関与できると分かる
* 詳細ルールで正確な数値を確認できる
