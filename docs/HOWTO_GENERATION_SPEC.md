# ONI PIN — 遊び方文案生成用・実装準拠仕様書

> **用途:** GPT 等に渡して「遊び方」UI向けの玩家向け説明文を生成するための、**現行コード準拠**の網羅仕様。  
> **更新基準:** `lib/game/game_config.dart`, `lib/game/match_duration_scaling.dart`, `lib/features/game_map/match/*`, `lib/game/werewolf_*.dart`  
> **注意:** 旧ドキュメント `GAME_DESIGN_SPEC.md` / `LOCATION_REVEALS.md` の一部（40秒定期・感染=名前付き暴露・鬼エリア外即勝利など）は**現行実装と不一致**の箇所あり。本書を優先すること。

---

## 1. ゲーム概要

- 屋外 GPS タッグ + 人狼要素の非対称対戦。
- **ライブ位置ピンは基本なし。** 断片情報（名前付き暴露・匿名痕跡・テキスト手がかり）を読んで追う・逃げる。
- 個人の脱落と陣営の勝敗は別。脱落後も第二ゲームで関与可能。
- 試合時間: 準備画面で **10〜90分**（既定 45分 = 2700秒）。`MatchDurationScaling.minMatchSeconds` = 600秒。

---

## 2. 勝敗条件（陣営）

### 2.1 人陣営の勝ち

1. **制限時間終了時**に人側生存者が 1人以上（`GameState.runnerWin` / `MatchEndReason.timeUp`）
2. **告発成功**（3人以上・設定による。`MatchEndReason.accusationSuccess`）
3. **鬼陣営の本鬼が 0人**（`WerewolfFactionLogic.countAliveFactions` で `oniAlive == 0` かつ `humanAlive > 0`）
   - 本鬼（`PlayerRole.hunter`）の捕獲・エリア外脱落など
   - **人狼だけが鬼陣営に残っていても、本鬼がいなければ人陣営勝利**（ゲーム破綻防止）

### 2.2 鬼陣営の勝ち

1. **人側生存者が 0人**（全員捕獲等。`MatchEndReason.allHumansEliminated`）
2. 告発に成功されず時間切れ前に人側を追い詰める

### 2.3 試合中止

- 参加者過半数の投票、またはホスト操作で `MatchEndReason.hostAbort`
- **勝敗・戦績・トロフィーは付与しない**
- **軌跡同意済みならギャラリーに「試合中止」として保存**（`endReason: host_abort`）
- 投票提案の回答期限: **60秒**（`abortProposalTimeoutSeconds`）

## 4. プレイエリア

### 4.1 判定

- **円:** 中心 + 半径。枠外 = 中心からの距離 − 半径 > 0
- **多角形:** 内部 / 外部。外部時の `overflow` = 辺までの最短距離

### 4.2 エリア外ルール（逃走者・鬼ともに適用）

| 段階 | 条件 | 効果 |
|------|------|------|
| 猶予 | 枠外だが **25m以内**（`outsideAreaGraceMeters`） | タイマー進行なし扱い（`overflow <= 25m` でリセット） |
| 警告猶予 | 枠外 **8秒未満**（`outsideAreaGraceSeconds`） | まだ暴露・脱落なし |
| 初回暴露 | 8秒以上・安全地帯チャージなし | **名前付き**位置暴露（`triggerLocationReveal`） |
| 安全地帯回避 | チャージあり | チャージ1消費で暴露回避（鬼エリア外時はチャージ消費不可） |
| 再暴露 | 既に暴露済みの外出継続 | **25秒ごと**（`outsideAreaRepeatRevealSeconds`）に追加暴露 |
| 脱落 | 外出 **90秒**継続（`outsideAreaEliminationSeconds`） | 脱落（`outsideElimination`） |

**鬼のエリア外:** 即人陣営勝利ではなく**脱落**。本鬼0人で人陣営勝利（§2.1-3）。

### 4.3 試合開始時のエリア補正

- **円のみ:** ホスト現在地がエリア外かつ **80m以上**はみ出し → 中心を現在地へ移動（半径維持）
- **多角形:** 自動補正なし

## 5. 接近・捕獲（3系統）

**前提:** 逃走者が追跡対象として存在するときのみ（`_chaseTargetsPresent`）。鬼単独テスト除く。鬼位置不明時は GPS 距離判定スキップ（テストモード除く）。

### 5.1 距離スケール（プレイエリア連動）

基準半径 = 円の半径、または多角形の外接近似。

| 名称 | 計算 | 範囲 |
|------|------|------|
| 接触圏 | 半径 × 8% | **35〜95m** |
| 拘束円 | 半径 × 10% | **45〜110m** |
| パニック圏 | 半径 × 14% | **58〜115m** |
| スキル捕獲結界 | 固定 | **55m** |
| 直接捕獲 GPS | 固定 | **12m** |
| 警告距離 | 固定 | **120m** |
| 危険距離 | 固定 | **70m** |

### 5.2 系統① 接触圏 → 接触拘束 → 捕獲

```
接触圏内に入る
  → 3秒: 逃走者に「接触圏内」警告（touchLockRunnerNoticeSeconds）
  → さらに4秒留まる（合計4秒以上）: 接触拘束開始
  → 拘束時間: 拘束円半径 ÷ 2.8m/s + 6秒 → 22〜75秒にクランプ
  → 拘束中:
      - 拘束円内にいれば安全
      - GPS 12m 以内 or BLE contact 帯 → 即捕獲
      - 拘束円外に出る → 10秒猶予（bindZoneEscapeGraceSeconds）
          - 10秒以内に戻れば OK
          - 10秒超: 位置暴露 → 捕獲
  → 拘束時間切れ: 拘束解除
```

- BLE `ProximityBand.contact` は GPS より優先的に脅威度が高い
- BLE 接触のみで GPS が遠い場合、鬼陣営人狼（捕獲不可）だけがいると BLE 単独捕獲をブロックする場合あり

### 5.3 系統② パニック（コード: infection）

**接触拘束とは別タイマー。脱落しない。痕跡は匿名。**

```
パニック圏内（接触圏より外側になりやすい）
  → 6秒連続（infectionExposureSeconds）でパニック状態開始
  → 警告: 入った直後 / 直前（imminent）
  → パニック持続: 22秒（infectionDurationSeconds）
  → パニック中: 7秒ごと（infectionRevealIntervalSeconds）に匿名痕跡
  → 偽位置スキル中は痕跡位置がデコイ側になることがある
```

| 定数 | 値 |
|------|-----|
| 発動まで | 6秒 |
| 持続 | 22秒 |
| 痕跡間隔 | 7秒 |
| 痕跡種別 | **匿名**（`source: panic`） |

### 5.4 系統③ スキル「捕獲結界」

| 定数 | 値 |
|------|-----|
| 半径 | 55m |
| 持続 | 24秒 |
| CD | 80秒 |
| 設置操作 | スキル押下 → 地図長押しで範囲確認 → 離して設置。×へドラッグでキャンセル |
| 円外10秒 | 暴露 → 捕獲（`bindZoneEscapeGraceSeconds`） |

- 逃走者・鬼どちらも装備可
- 接触拘束の `lockZone` と共有ランタイムだが `lockZoneFromSkill=true` で区別

### 5.5 その他スキル（接近系）

**体投げ（鬼）:** 射程90m・人形12秒・CD75秒・地図タップ猶予22秒。置けなければ発動位置が暴露。

**偽位置（逃走者）:** 20秒間・CD72秒。進行方向へ38mオフセット、3.4m/s でドリフト。暴露はデコイ座標へ。

**偽情報暴露（鬼）:** CD75秒。名前付きの偽暴露。

|----------|------|

## 8. 告発システム（3人以上）

### 8.1 解禁

**どちらか早い方:**

1. 試合経過 **≥ 試合時間 × 60%**（`accusationUnlockTimeRatio = 0.6`）
2. 脱落 **≥1人** かつ 経過 **≥ max(5分, 試合25%)**、上限15分（`MatchDurationScaling.accusationUnlockMinElapsedSeconds`）

例（45分試合）: 60% = 27分、脱落+経過 = 11分15秒

### 8.2 有効施設数

- 解禁前: **0**
- 解禁時: **1**
- 残響体陣取り: +1（試合 **2回**まで）
- 復讐の鬼影妨害: −1（試合 **3回**まで）
- 脱落数だけでは増えない

### 8.3 告発者・対象

- **生存中の逃走者のみ**（`PlayerRole.runner`）。人狼・鬼・脱落者は不可
- 対象が本鬼（`PlayerRole.hunter`）かどうかで成否

### 8.4 告発の重み（ホスト設定）

| モード | 正解 | 失敗 |
|--------|------|------|
| 即勝利（標準） | 人陣営即勝利 | 告発者即脱落→残響体 |
| 鬼を脱落 | 本鬼脱落・試合継続 | 告発権消費のみ |
| ポイント | 人陣営+1点（終了時集計） | 告発権消費のみ |

## 9. 役職・スキル

### 9.1 装備数

| 役職 | 候補スキル | 装備数 |
|------|-----------|--------|
| 逃走者 | 偽位置、捕獲結界 | 1 |
| 鬼 | 偽情報暴露、捕獲結界、体投げ | 2 |
| 人狼 | 鬼化（変身） | 1 |

### 9.2 逃走者特化（3人以上）

- **アナリスト:** 匿名痕跡に時間帯・源・信頼度の読み取り補助（名前は出さない）
- **ハッカー:** 情報屋の距離帯精密化 + 鬼の方角（座標ピンなし、最低移動8m）

### 9.3 人狼

**陣営（脱落時点で固定）:** 自分以外の生存者の人ロール数 vs 鬼ロール数

- 人ロール ≤ 鬼ロール → **人陣営**（鬼化中に捕獲可）
- 人ロール > 鬼ロール → **鬼陣営**（鬼化中は拘束・パニックのみ、捕獲不可）

**変身スケジュール:**

- 強制切替間隔: `min(600秒, 試合時間÷3)`（前回切替から）
- 任意切替CD: 強制間隔 × 0.75
- 鬼同士（本鬼+鬼化人狼）はパニック・捕獲しない
- 告発不可

### 9.4 オンライン役職配分

- 2人以下: 最低1逃走者・1対立役
- 3人以上ランダム: 鬼1・人狼1（6人以上は人狼2）・残り逃走者
- 人数指定モード: ホストが鬼/人狼数を指定

## 10. 第二ゲーム（脱落後）

**分岐:** 試合既定が残響体モード時、脱落時点の陣営で自動分岐（人→残響体、鬼→復讐の鬼影）

### 10.1 残響体（人側脱落）

| 行動 | チャージ | 個人CD | 試合上限 |
|------|---------|--------|---------|
| 監視端子ジャック（鬼位置暴露） | 15秒 | 100秒 | 5回 |
| 告発施設陣取り（有効+1） | 16秒 | 90秒 | 2回（告発解禁後） |

### 10.2 復讐の鬼影（鬼側脱落）

| 行動 | チャージ | 個人CD | 試合上限 |
|------|---------|--------|---------|
| 告発施設妨害（有効−1） | 18秒 | 90秒 | 3回 |
| カメラ停止（各カメラ1回） | 14秒 | 25秒 | 無制限（カメラごと1回） |

### 10.3 その他脱落後ルール

- **幽霊:** 観戦・粗位置のみ
- **鬼側合流:** 索敵支援（粗位置）

## 11. 鬼の情報（追跡側）

- **ライブ位置共有なし**（本鬼は Firestore へ3秒間隔で位置送信するが、逃走者には常時ピンではない）
- **遅延軌跡:** 試合の22〜38%前の帯（試合時間連動）
- **序盤アンカー:** 試合開始付近の鬼位置ヒント（試合15%経過まで、最長10分）
- **情報屋モード**（ホスト設定）: 方角のみ / 距離帯のみ / 断片（12秒ごと切替）

## 13. 記録・ギャラリー

- 軌跡保存は**ユーザー同意**が必要
- 保存内容: プレイヤー軌跡、暴露ログ、イベント、プレイエリア、ギミック配置
- 単独逃走で鬼軌跡は記録しない（東京デフォルト座標の誤表示防止）
- 試合中止: `endReason=host_abort`、見出し「試合中止」
- 軌跡サンプル間隔: 最小2秒、簡略化8m/点

## 14. 遊び方に載せていないが実装されている細部

- GPS ジャンプ無視: 2秒以内に120m超の移動
- 暴露ログ最大50件（ローカル）
- マッチイベント最大120件
- 鬼単独ではパニック・接触ループは走らない
- 偽位置中の暴露はデコイ座標に寄せる
- オンライン再接続: `startedAt` から経過時間復元
- オフライン準備: 試合時間は SharedPreferences に永続化
- コーチマーク・チュートリアル・ガイドハブは別UI
- テストモード: デバッグビルドのみ、鬼位置未知でも距離判定可

## 15. 文案生成時のトーン指針

- 玩家向けには「感染」ではなく **「パニック」** を使う（`MatchUiTerms`）
- 秒数は「おおよそ」「約」でよいが、上表の順序関係（接触圏 < パニック圏、エリア外90秒脱落など）は崩さない
- 匿名と名前付きの違いを必ず区別する
- 鬼エリア外は「即勝利」ではなく「鬼の脱落→本鬼0で人陣営勝利」
- 準備・エリア編集・ルーム操作の詳細は遊び方シートのスコープ外（別コーチマーク）
