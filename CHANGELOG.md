# Changelog

All notable changes to ONI PIN are documented here.

## [Unreleased]

### 整理（完了ベースライン 2026-07）

- **match_events part** — 試合イベント送信・定期匿名・reveal/gimmick publish を skills から分離。
- **本体フィールド**を Online / Elimination / Progression 等にセクション分け。
- **索引**を 15 part に更新し、「ここで一区切り」と明記（HANDBOOK 同趣旨）。
- 段階1〜2の SSOT・告発 outcome・capture_zone・HUD/play_area 分離・MemberRoleWire を含む。

### 整理（段階2: part 責務・用語）

- **skills ジャンク撤去** — トースト／条件 HUD → `hud_experience`、エリア編集 → `play_area`、告発ブロック判定 → `accusation`。
- **`MemberRoleWire`** — `oni`/`hunter` ワイヤ判定と表示ラベルを `lib/sync/` に集約。
- **online_sync** 先頭にイベント→適用 part 対応表を追加。
- 索引・CHANGELOG を同期。

### 整理（段階1: 地図・重複削減）

- **GameMapScreen 索引を現行 14 part に更新**（`capture_zone` / `host_light` / `prep_sync` / `presentation` / `rejoin`）。
- **仕様 SSOT** — HANDBOOK / PLAYER_REFERENCE / how_to_play_v2 に優先順位とアーカイブ明示。
- **告発結果を `accusation_outcome.dart` に集約** — host / local / host-light が同じ分岐を参照。
- **捕獲結界を `game_map_screen.capture_zone.dart` に分離**（skills ジャンクドロアから切り出し）。
- **GuideTerms → MatchUiTerms 参照**で重なる表示文言を一本化。
- **`MatchGeoHelpers.captureZoneTargetIds`** を抽出してテスト可能に。

### バグ修正（致命〜高）

- **告発解禁で有効施設が 0 のまま** — 解禁ペイロード生成時に `treatAsUnlocked` で施設を選び、空 indices は受信側で再計算。
- **鬼前面の捕獲が拘束なしで発火** — ホストライト捕獲を「拘束中 + GPS12m/BLE」に揃え、bound を結界持続で期限切れに。
- **捕獲結界の致死フラグ欠落** — `capture_zone_bound` に `capturePermitted` を載せ、鬼陣営人狼の非致死結界が viewer 側で捕獲可にならないよう修正。
- **告発権だけ消えて解決されない** — 送信直後は判定待ち、解決で消費確定。ホスト不通時は引継ぎ／参加者救済で解決。
- **円エリアの 80m 自動寄せが未配線** — 試合開始検証で `_alignPlayAreaToCurrentPositionIfFar` を実行。
- **ビルド破綻** — `WorldProfile.urbanHorror` → `horror`、`unawaited(void)` を除去。

### 同期・試合開始

- **試合開始フローを短縮** — 役職カード（自動クローズ）→ カウントダウン → 試合開始に統一し、`startedAtUtc` を実際の開始タイミングに合わせる構成へ。
- **同期開始タイミングを整理** — `GameState.running` に入ったあと `_armMatchSync` を有効化し、演出中のズレを抑制。
- **オンボーディング非ブロッキング化** — 試合開始後の案内・コーチマークは遅延表示にして、試合処理を止めないよう調整。
- **開始失敗時のロールバック強化** — `publishMatchStart` 失敗時に待機状態へ安全リセットして中途半端な試合状態を残さない。

### 座標同期・近接

- **presence 間隔を短縮** — 通常 6 秒 / 追跡 3 秒（従来 12 秒 / 5.5 秒）。
- **鬼位置フォールバック** — `hunter_position` が遅れた場合も `members.lat/lng`（fresh のみ）で近接判定を補助。
- **結界 rescue の発火判定改善** — 非ホスト救済は配置時ではなく発火時にホスト不在を判定し、取りこぼしを軽減。
- **結界 rescue の再試行を追加** — ホスト状態の瞬間変化で bound が落ちるケースに対し短い再判定を行う。
- **告発施設ブロック** — 本鬼位置を presence / events の解決座標で判定。
- **第二ゲーム** — 試合タイマー 0 付近でも残響体／鬼影チャージ評価を継続。

## [3.1.0] — 2026-06-29

### プレイ体験・文言

- **パニック・拘束の日本語** — 「足が止まりやすく」等をやめ、パニック／拘束の実際の挙動に合わせて統一（`match_hud_copy.dart`）。
- **遊び方ガイド** — 接触拘束解除後に捕獲結界で拘束し直し・結界中の至近捕獲の流れを追記。
- **名前付き暴露の写真** — `subjectUid` を Firestore reveal に載せ、鬼の情報屋でも写真ピンが他端末に届きやすく調整。
- **ギミック再配置** — 安全地帯・情報屋の移動先を試合開始時と同様に道路上へスナップ（API キーがある場合）。

### 可読性・オンボーディング（v3.0 以降の積み上げ）

- 世界観別ダイアログ・HUD・メニューの文字色を監査・修正。
- 通話プレイの案内を「ONI PIN を先に起動し、通話はバックグラウンド」に統一。
- 人狼 HUD: **自動切替まで** / **切替可能まで**（旧「強制まで」「切替CD」）。

### ドキュメント

- `PLAYER_REFERENCE.md`・`GAME_DESIGN_SPEC.md`・実機チェックリスト等を現行実装に同期。
- バージョン表記を `pubspec.yaml` と `lib/app_version.dart` で一元管理。

### リリース時の注意（運用）

- **Firestore ルール** — 本リリースはルール変更なし。3.0.0 以降未 deploy の場合のみ `firebase deploy --only firestore:rules`。
- **iOS / Android** — `version` は `3.1.0+4`（ビルド番号 **4** を App Store / サイドロード用に必ず増分）。

## [3.0.0] — 2026-06-16

### 通話中プレイ（Host-Light）

- **非ホスト救済** — ホストが通話アプリ前面（background / stale）のとき、時間切れ・告発解禁・結界 bound・切断脱落・陣営全滅を参加者が冪等イベントで補助。
- **鬼前面の捕獲イベント** — 逃走者が background でも、鬼端末が `oni_capture_elimination` を発行可能（復帰時に反映）。
- **バックグラウンド近接判定** — オンライン試合中、通話アプリ前面・画面ロック中も近接・捕獲・パニックと危機通知を継続（スキル・告発 UI は前面復帰が必要）。
- **遊び方・試合前ヒント** — 通話プレイの説明を `guide_sections` と試合前ダイアログに追加。

### スキル・遊び方・UI

- **装備スキル一次ソース** — `lib/game/skill_reference.dart` に5スキルの文案・数値表を集約。遊び方・チュートリアル・役職ダイアログと同期。
- **人狼 HUD** — 「強制まで」（自動切替までの秒数）と「切替CD」（任意切替までの秒数）を別チップ表示。強制切替直後は切替CDがやや長い。
- **チュートリアル → スキルカード直リンク** — 偽位置・捕獲結界・鬼化ステップから遊び方の該当カードへジャンプ。
- **リプレイ軌跡ラベル** — 「鬼（ローカル）」を「鬼」に変更（技術用語を排除）。
- **ワールドギャラリー** — 見出しは WORLD GALLERY＋日本語補足。確定ボタンは世界観ごとの英語（和風のみ日本語）。試聴チップは操作音/暴露音/捕獲音/BGM。
- **ドキュメント統合** — [docs/PLAYER_REFERENCE.md](docs/PLAYER_REFERENCE.md) をプレイヤー向け・リリースノート用の現行仕様に。`how_to_play_v2` は設計メモ扱いに整理。

### 注意

- `firestore.rules` の救済イベント追加は **Firebase deploy が必要**。

## [2.0.0] — 2026-06-06

### ゲーム体験

- **試合プリセット（お手軽 / 標準 / じっくり）** — ルーム作成直後に3択。時間・エリア・ギミック密度を一括設定。詳細は後からカスタムメニューで調整可能。
- **告発重みモード** — 即勝利 / 鬼脱落 / ポイント加算 をホストが選択。
- **HUD フェーズ表示** — 隠密逃走 / 告発解禁 / 残りN分 を一行表示。
- **試合イベント履歴** — 告発解禁・脱落・告発成否などをトースト + HUD 1行で表示。
- **スキル長押し CD 表示** — 試合中、長押しで残りクールダウンを常時表示。
- **第二ゲーム導線強化** — ウェルカムに脱落後プレイ説明、初回脱落時3秒オーバーレイ、使える施設/カメラのハイライト（使えないものは薄表示）。
- **準備・ロビーのルール可視化** — ホスト向け設定サマリ一行、非ホスト向けルール概要カード。

### コード品質

- **`game_map_screen.dart` を part ファイルに分割** — オンライン同期・暴露・HUD・エリア編集・試合ライフサイクル・告発・第二ゲーム・スキル・オーバーレイの9ドメイン。索引: `lib/features/game_map/game_map_screen_index.dart`。
- **静的解析クリーン** — `flutter analyze lib` エラー・警告 0。
- **テスト拡充** — 告発重み、設定サマリ、HUD フェーズ、プリセット、進行・役職ミックス等。

### オーディオ

- 世界観別 BGM / 環境音アセット整備（`assets/audio/`）。
